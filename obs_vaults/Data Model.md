# Data Model

Purpose: Describe the persistent entities used by NeoChat.

Related: [[Home]], [[Architecture]], [[apis/supabase-tables]], [[modules/sqlite-db]]

## Entities

- `chats`: Workspace records with an `id` and `created_at` timestamp.
- `messages`: Chat messages linked to `chats` by `chat_id`.
- `storage.objects`: Supabase-managed object rows for uploaded images in bucket `chat-images`.

## Supabase Schema

`supabase/schema.sql` defines `chats.id` as a UUID with `gen_random_uuid()` and `messages.id` as an identity `BIGINT`.

Messages include:

- `chat_id`: References `chats(id)` with cascade delete.
- `sender_id`: Text browser identifier.
- `type`: `text` or `image`.
- `content`: Message body or image URL.
- `created_at`: Timestamp with time zone.

## SQLite Schema

`server/db.js` defines a similar local schema in SQLite.

- `chats.id` is text and generated by `crypto.randomUUID()` in `server/index.js`.
- `messages.id` is an autoincrement integer.
- `messages.type` defaults to `text` but has no explicit SQLite check constraint.

## Access Policy

`supabase/schema.sql` enables RLS, then creates permissive policies for anonymous users on `chats`, `messages`, and storage objects in `chat-images`.
